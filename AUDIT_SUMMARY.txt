  Field:              Salary
  Entity Type:        float
  Database Column:    DECIMAL(18,2)
  Stored Proc Param:  @Salary DECIMAL(18,2)
  Severity:           🟠 HIGH
  Impact:             Financial data precision loss during calculations
  Root Cause:         float uses IEEE 754 binary representation; DECIMAL uses base-10
  
  Precision Problem:
    Salary in DB: 2500.50 (exactly 2 decimal places)
    Read as float: Could be 2500.4999999999 or 2500.5000000001
    Accumulated errors in multiple calculations
    
  Financial Impact:
    - Over thousands of employees and years
    - Compounding rounding errors
    - Audit trail discrepancies
    - Banking compliance violations
    
  Fix: Change Employee.Salary from float to decimal

How the mapping works:
1. DbAccessService reflects over entity properties
2. For each property, creates @PropertyName parameter
3. Passes entity object to Dapper
4. Dapper maps properties to parameters by name (case-insensitive)
5. Dapper converts C# types to SQL types
